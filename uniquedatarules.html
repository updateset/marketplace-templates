Unique Data Rules

The Unique Data Rule application allows you to quickly create logic to prevent duplicate data from being inserted into your tables in ServiceNow.

The out of the box method to specify a unique field only allows for 1 field to be selected, however, our business cases may not alwayas be that simple.

The best part of the Unique Data rule application is you can have a central location where all of your composite database keys can be viewed from a central location.


Usage

Role Required: admin

Navigate to Unique Data > Unique Data Mappings and start by creating a new record.

On form there are 5 fields:

  • Fields
    • Rule Name - Provide a human readable name for the unique key
    • Active - allows you to disable the condition withouth having to delete it.
    • Table - a drop down reference to the table you wish to apply the rules to
    • Non Dupelicate Fields - select the fields you wish to use in your rule, these fields are driven by the table seleceted
    • Composite Key - a true/false checkbox that drives the logic on the unique record detection
      • True - Each field in the Non Duplicate Fields will be treated as an AND condition i.e if name and email is selcted then both fields must have the same value for the error to fire and prevent the record from being generated
      • False - Each field in the Non Duplicate Fields will be treated as an OR condition i.e if name and email is selected then if a new record is inserted with just the same name then the error will fire to prevent the record from being generated

When a new record is generated a new Business Rule is added to the table selected that performes the logic necessary to catch the duplicate fields

If you add multiple entries on the same table they will be treated as an OR condition

For example if you have 2 rules on the sys_user_group table one where its a composit rule on the name and type and another where its not a compisit rule on group email and manager

The end result will be a duplicate will fire any time these conditions are met

  1. A group with the same name and type
  2. a group with the same email address
  3. A group with the same manager

Installation

Procedure

  1. Navigate to All > System Update Sets > Retrieved Update Sets and open the update set.
  2. Use the related link Import Update Set from XML and select the XML downloded form the GitHub Repo.
  3. Search for Update Set name Unique Data Rules and click it.
  4. Preview the Update Set
  5. Click Commit Update Set.
    • Click Cancel to return to the preview and reevaluate the change. None of the updates are committed.
    • Click OK to skip the change and continue committing the changes that are marked as Commit.
    If the update set contains one or more DELETEs for schema, the system displays a warning. The warning lists up to five updates that may contain problems. If more than five updates have potential problems, the system provides a link.

    When the system successfully commits an update set, it displays a completion page.

  6. [Recommended] Click Commit log on the confirmation page, or navigate to System Update Sets > Update log and filter for the update set name.
    • Look for warnings that contain the text unsafe edit. The system automatically skips any changes that results in data loss, such as changing the type of a field that contains data. You must manually make any of these changes, if necessary. Use caution when making changes that affect production data.
    • Look for errors that indicate which records failed to commit and why. Create a new update set to address those failures, if necessary.
